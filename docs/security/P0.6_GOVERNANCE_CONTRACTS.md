# Governance Enhancements — Onda 5 (Contracts Only)

**Status:** Additive scaffolding · Architecture Freeze v1.0 COMPATIBLE
**Location:** `src/lib/ai/governance/` (types + in-memory reference impls)

Public contracts for the 10 governance primitives the Brain / Rules Engine /
Planner / Recommendation Engine will consume in later waves. Nothing here
touches Provider Layer, RLS, DB schema, or public API contracts.

## Invariants

- **Immutability:** `RuleRegistry.register` refuses to overwrite an existing version (fingerprint-locked).
- **Determinism:** All fingerprints via SHA-256 short (16 hex) — same input → same output.
- **Purity:** `evaluateConfidence` is a pure function; no I/O, no globals.
- **Additive:** No existing module imports governance yet — features opt in on their own timeline.
- **Multi-tenant:** Every trace/snapshot/lineage node carries `organizationId`; stores are per-instance and never share state across tenants at persistence layer (future).

## Extension Points for Onda 5+

- Persist `DecisionTrace` in a new (RLS-protected) `ai_decision_traces` table.
- Persist `ContextSnapshot` bodies in KV / R2 with the fingerprint as key.
- Persist `ArtifactLineageNode` edges in `ai_artifacts` + `ai_artifact_edges`.
- Wire `ruleRegistry` to a DB-backed `ai_rules` table (still versioned + immutable).
- Wire `evaluateConfidence` into the post-processor before responses ship.

None of the above requires touching frozen components.
